# SemiPLM 产品通用规范

本文档约束 SemiPLM 的产品边界、业务对象、状态流转、数据来源、权限控制、界面交互和文件处理。新增功能、改版页面、接口调整和数据初始化都必须对照本文档执行。

---

## 一、产品定位

| 规则 | 约定 |
|---|---|
| 产品目标 | 面向半导体制造企业的轻量级单机商用 PLM，不做演示玩具页 |
| MES 对齐边界 | PLM 主控产品定义、工艺流程（含工序/制程内容/参数/量测/QTime/防污染/动作/分支）、用料表、ECN 发布和 MES 同步包；MES 主控生产执行、设备运行态、设备配方主体和 Lot/Wafer 过站 |
| 当前组织边界 | 单公司、单工厂、单组织；部门仅作为用户属性、流程路由和统计维度 |
| 开发优先级 | 真实对象、真实 CRUD、真实状态、真实关联、真实审批、真实集成队列优先 |
| 暂缓范围 | 多租户、多法人、多站点、高并发、复杂事务、CAD/EDA 深集成、移动端暂缓 |
| 页面风格 | 浅色、高密度、低干扰的企业后台风格，优先保证业务可用性 |
| 禁止事项 | 禁止只做静态展示页；禁止出现“演示环境”“当前仅支持”等实现口径文案 |

## 二、业务对象通用规则

| 规则 | 约定 |
|---|---|
| 对象边界 | 每个功能必须有明确业务对象和对象边界，不能做孤岛页面 |
| 唯一标识 | 关键对象必须有业务编号，编号规则来自编码规则或后端生成 |
| 基础字段 | 关键对象至少包含编号、名称、状态、负责人、创建人、创建时间、更新时间 |
| 生命周期 | 文档、BOM、工艺、变更、项目、质量等核心对象必须有状态或生命周期 |
| 对象关联 | 产品、文档、BOM、工艺、变更、项目、质量之间必须能互相追溯 |
| 详情聚合 | 详情页优先聚合关联对象、文件、流程、日志，不把能力拆成过多菜单 |
| 产品详情边界 | 产品详情展示 ProductDef 基础信息、关联工艺流程版本、关联 BOM 版本、MES 映射和同步记录；流程工序/内容/量测/防污染等在工艺流程详情维护 |
| 历史追溯 | 发布、审批、归档、作废、关闭等关键动作必须保留历史记录 |

## 二点一、半导体产品建模边界

SemiPLM 的产品、工艺、用料表结构对齐 MES 制造建模，保证发布后能同步给 MES。8 模块（Seq/Content/Parameter/Measure/QTime/Contamination/Action/Alter）挂在工艺流程详情页的 tab，不单独建菜单、不挂在产品详情页。产品详情页只展示关联的工艺流程版本和用料表版本。

| PLM 功能 | 对齐 MES 模块 | PLM 边界 | 挂载位置 |
|---|---|---|---|
| 产品基础信息 | `ProductDef.basicInfo` | 主控产品名称、版本、产品类型、生产类型、产品组、状态、Owner、Gross Die、Bank、光罩、Bin、Dummy 策略 | 产品中心 > 产品库 > 详情页 |
| 用料表 | `bomName` / `bomVersion` / BOM 专用接口 | 主控设计用料、版本、关键物料、替代料、生效范围；MES/ERP 负责库存、领料、消耗 | BOM 管理 > 设计 BOM |
| 流程工序 | `Product.detail type=Seq` / `seqLo` | 主控工序序列、工步名称、工步版本、工艺段、分段、工艺层 | 工艺管理 > 工艺流程 > 工序 tab |
| 流程内容 | `Product.detail type=Content` / `content` | 引用 PLM 自有的工艺能力/工艺配方、光罩/探针卡、抽检规则、跳站/强制工步 | 工艺管理 > 工艺流程 > 制程内容 tab |
| 动态参数 | `Product.detail type=Parameter` / `parameter` | 主控参数名、属性、参数值 | 工艺管理 > 工艺参数 |
| 量测计划 | `Product.detail type=Measure` / `measure` | 主控量测项、目标值、上下限、抽检数量、抽检卡槽、抽检方式 | 工艺管理 > 工艺流程 > 量测 tab |
| QTime | `Product.detail type=Qtime` / `qTime` | 主控起止工序、事件、限制分钟、最大 WIP、报警引用 | 工艺管理 > 工艺流程 > QTime tab |
| 防污染 | `Product.detail type=Contamination` / `contamination` | 主控进站允许污染等级和出站污染等级 | 工艺管理 > 工艺流程 > 防污染 tab |
| 动作 | `Product.detail type=Action` / `action` | 受控维护进出站动作、原因码和变更工步，第一阶段谨慎开放 | 工艺管理 > 工艺流程 > 动作 tab |
| 分支/返工 | `Product.detail type=Alter` / `alter` | 受控维护分支、返工路径、返回工步和最大返工次数，第一阶段谨慎开放 | 工艺管理 > 工艺流程 > 分支 tab |

工艺流程是 PLM 主控源头，设计、审批、发布后同步给 MES。标准工序（ProcessStep）、工艺阶段（ProcessStage）、工艺能力（ProcessCapability）、工艺配方（Recipe）、设备类型（EquipmentType）和设备能力（EquipmentCapability）为 PLM 主控设计层对象；其中 EquipmentCapability 按 `equipmentTypeName + processCapabilityName` 建模，用于工艺设计阶段按设备类型校验能力。设备配方（EquipmentRecipe）、设备配方参数（EquipmentRecipeParam）、炉管配方（FurnaceRecipe）和物理设备实例属 MES 执行层，PLM 只做引用、校验、对账。

## 三、状态与动作

| 规则 | 约定 |
|---|---|
| 状态来源 | 状态来自生命周期、流程配置或数据字典，禁止页面随意硬编码 |
| 状态机 | 有生命周期的对象必须定义允许的状态流转 |
| 后端兜底 | 状态流转、发布、审批、归档、作废、删除必须由后端统一校验 |
| 前端展示 | 前端按钮显隐/禁用必须与后端规则一致；条件复杂时优先由后端返回可用动作 |
| 不可逆动作 | 删除、发布、归档、作废、关闭等不可逆动作必须二次确认 |
| 删除边界 | 已发布、已审批、已归档或已被其他对象引用的数据禁止物理删除，优先作废/停用 |
| 操作反馈 | 成功显示明确反馈，失败显示后端 `detail`，不能吞错误 |
| 审计日志 | 增删改、提交、审批、发布、归档、作废、关闭、重试等关键动作写 OperationLog |

## 四、数据来源与主数据

| 规则 | 约定 |
|---|---|
| 主数据字段 | 负责人、执行人、审核人、角色、产品、物料、文档、BOM、工艺、项目等必须来自受控数据源 |
| 人员选择 | 人员字段使用受控用户选择组件，禁止自由手输 |
| 枚举来源 | 类型、状态、优先级、严重度、风险等级等下拉选项从数据字典加载 |
| 字典维护 | 新增字典在“基础平台 > 数据字典”维护，不为单个页面改代码 |
| 字典 code | 使用 `DICT_` 前缀 + 大写下划线，如 `DICT_PRIORITY` |
| 字典缓存 | 前端使用模块级缓存，多组件共享，避免重复请求 |
| 硬编码限制 | 允许少量固定技术常量，但业务选项、状态、类型不得散落在页面代码里 |

## 五、接口与列表契约

| 规则 | 约定 |
|---|---|
| API 归属 | 后端业务 API 写入对应 `routers/{domain}.py`，前端 API 写入 `frontend/src/api/{domain}.ts` |
| 列表返回 | 分页列表统一返回 `{ items, total, page, page_size }` |
| 分页参数 | 统一使用 `page`、`page_size`，默认 `page_size=20` |
| 搜索参数 | 统一使用 `keyword`，按业务关键字段模糊匹配，不搜全字段 |
| 排序参数 | 统一使用 `sort_by`、`sort_order`，分页列表必须后端排序 |
| 筛选参数 | 筛选字段命名与后端模型或业务语义一致，避免页面自造别名 |
| 新增后 | 列表刷新到第一页，新增数据出现在第一条 |
| 编辑后 | 保持当前页和筛选条件，仅更新对应行或刷新当前页 |
| 批量操作 | 必须显示已选数量；部分成功时展示成功/失败明细 |

## 六、权限与安全

| 规则 | 约定 |
|---|---|
| 权限判断 | 前端使用统一权限函数控制按钮显隐/禁用 |
| 后端校验 | 写接口、状态流转接口、文件接口必须做后端权限校验 |
| 无权限反馈 | 403 显示无权限提示；业务条件不满足显示具体原因 |
| 禁用说明 | 因状态、引用、权限导致按钮不可用时，优先提供 tooltip 或错误说明 |
| 数据越权 | 用户只能访问其权限范围内的数据，不能只靠前端筛选 |
| 审计范围 | 涉及权限、审批、发布、文件、集成重试的操作必须可追溯 |

## 七、页面与导航

| 规则 | 约定 |
|---|---|
| 菜单归属 | 菜单按业务域组织，不按页面数量凑菜单 |
| 菜单入口 | 只开放已有真实页面，不放空壳入口 |
| 内部能力 | 版本、签核、发布、发放、回收、预览、权限等作为对象内部能力，不单独拆菜单 |
| 页面结构 | 业务页以列表、筛选、详情、操作区、状态反馈为主 |
| 详情页 | 顶部展示编号、名称、状态、负责人、更新时间和关键动作 |
| 详情 tab | 常用顺序为基础信息、关联对象、文件附件、流程记录、操作日志 |
| 跨对象跳转 | 关联对象显示可点击编号或名称，支持跳转到对应详情 |

## 八、通用界面规则

| 规则 | 约定 |
|---|---|
| 组件策略 | 当前允许 Element Plus 与 Arco 并存，规则优先描述产品行为，不绑定单一组件库 |
| 表格密度 | 高密度业务表格优先保证扫描效率，不使用斑马纹 |
| 默认排序 | 新增数据默认排第一，后端使用 `id DESC` 或 `created_at DESC` |
| 列宽 | 编号、状态、日期固定宽度；名称、描述使用 min-width 自适应 |
| 文本溢出 | 长文本不撑高行，使用 tooltip 或等价方式展示完整内容 |
| 空状态 | 显示“暂无数据”图标和文案，不显示空表格骨架 |
| 加载态 | 加载覆盖当前数据区域，不遮挡整页导航和页面结构 |
| 操作列 | 表格操作列固定在右侧，单行排列，不撑高行 |
| 操作顺序 | 主操作、次操作、查看类、危险操作；删除等危险操作放最后 |
| 更多操作 | 高频操作平铺，低频操作收进“更多”下拉；危险操作前加分割线 |
| 状态展示 | 状态使用标签组件展示，不使用纯文本 |
| 状态颜色 | 成功=绿色，警告=黄色，危险=红色，中性=蓝灰；同一状态全产品颜色一致 |

## 九、搜索、筛选与排序

| 规则 | 约定 |
|---|---|
| 搜索触发 | 回车搜索，清空自动搜索，不做实时搜索 |
| 搜索范围 | 搜编号、名称、型号、关键业务字段，不做全字段扫描 |
| 筛选下拉 | 筛选项来自数据字典或受控数据源 |
| 重置行为 | 重置后清空筛选和关键词，回到第一页 |
| 排序行为 | 分页表格必须后端排序，禁止只排序当前页 |
| 条件保持 | 查看详情、编辑、返回列表后尽量保持分页、搜索和筛选条件 |

## 十、表单与弹窗

| 规则 | 约定 |
|---|---|
| 表单布局 | 弹窗内表单优先使用双列网格，复杂对象可使用 tab |
| 控件宽度 | 表单控件宽度统一铺满当前字段区域 |
| 跨列字段 | 描述、备注、原因、影响分析等长文本跨两列 |
| 标签宽度 | 常规标签统一 100px，长标签场景可调到 110-120px |
| 必填标记 | 必填字段显示红色 `*` |
| 日期字段 | 日期使用日期选择组件，格式统一 `YYYY-MM-DD` |
| 弹窗宽度 | 简单表单 560px，标准表单 640-680px，复杂 tab 720-760px，详情数据包 900px 以上 |
| 关闭方式 | 编辑弹窗点击遮罩不关闭，必须点取消/关闭 |
| 校验行为 | 点保存时校验，校验失败不关闭弹窗 |
| 保存状态 | 保存中按钮进入 loading，防止重复提交 |
| 表单重置 | 关闭弹窗时重置表单，避免下次打开残留旧数据 |

## 十一、文件与附件

| 规则 | 约定 |
|---|---|
| 入口统一 | 主文件与附件同一入口展示，主文件标注“主文件” |
| 上传条件 | 上传条件由对象状态和权限共同决定，前端禁用与后端拦截一致 |
| 文件存储 | 数据库只存受控文件名和元数据，运行时由后端拼接实际路径 |
| 文件命名 | 使用对象类型、对象 ID、唯一值和原始文件名组合，避免冲突 |
| 文件安全 | 上传文件名必须清洗，禁止路径穿越；下载必须校验对象权限 |
| 下载行为 | 有权限且文件存在时允许下载，返回 blob 并携带文件名 |
| 预览范围 | PDF 和图片支持内嵌预览，其他格式只下载 |
| 附件加载 | 展开行或详情 tab 触发加载，结果缓存到响应式对象，避免重复请求 |

## 十二、集成与外部系统

| 规则 | 约定 |
|---|---|
| 集成定位 | PLM 负责对象结构、版本、状态和发布结果，ERP/MES/QMS 负责各自执行域 |
| 队列优先 | 涉及 ERP/MES/QMS 的动作先进入集成队列，不直接假装已同步 |
| 同步记录 | 集成队列记录目标系统、对象类型、对象编号、动作、状态、失败原因和重试次数 |
| 失败处理 | 失败可查看原因并重试，重试必须写日志 |
| MES 产品建模 | MES 同步优先围绕 `ProductDef`、产品详情模块、BOM 绑定、ECN 生效和同步回写，不直接维护设备实例、设备配方主体、报警、批次执行 |
| 暂缓内容 | 字段级复杂映射、真实回调、接口日志平台等在实际对接时深化 |

## 十三、完成标准

一个功能点只有同时满足以下要求，才算完成：

- 有明确业务对象和对象边界。
- 有数据库表或明确复用的数据模型。
- 有后端 API，不只做前端静态页面。
- 支持必要 CRUD 或业务动作。
- 有状态、生命周期或业务阶段。
- 有权限控制和后端校验。
- 需要审批的对象接入流程模板或流程实例。
- 与相关对象有真实关联，不做孤岛页面。
- 关键动作有反馈、日志和可追溯记录。
- 涉及外部系统的动作进入集成队列。
- 页面符合高密度商用后台风格。
- 主数据字段使用受控数据源。
- 完成后验证后端编译、前端构建、关键接口和关键页面。
