# SemiPLM 功能清单

## 参考范围

本清单按通用商用 PLM 能力进行设计，结合用户提供的武汉睿码 PLM 功能截图作为主要落地参考，并以鼎捷 PLM 及 Siemens Teamcenter、PTC Windchill、Aras Innovator、Dassault ENOVIA、OpenBOM 等公开资料中的共性能力作为校准。目标不是复制任何第三方私有产品，而是实现光电芯片制造企业可落地的轻量单机 PLM。

## 产品约束与交互标准

- 以通用 PLM 主流能力为框架，满足光电芯片制造行业；武汉睿码功能清单为主参考，鼎捷为辅参考，厂商特有能力只在确有必要时吸收。
- 完整商用菜单和功能域以 `MENU_ARCHITECTURE.md` 为准；开发时先对齐菜单设计，再逐项实现功能。
- 所有功能必须对接后端和数据库，不能只有页面。
- 所有核心对象必须有真实 CRUD、生命周期状态、权限和流程/审批边界。
- 人员类字段必须来自用户主数据；角色类字段来自角色主数据；业务对象引用必须来自对应对象列表。
- 操作列采用紧凑单行按钮，不允许换行、撑高行高或使用大面积删除按钮。
- 删除、发布、关闭等关键动作必须有确认或业务校验。
- UI 风格采用浅色 Arco Design Pro 式企业后台密度，避免演示文案、部署文案和说明型卡片堆砌。
- 当前组织口径：南智光电 / 生产部；不做多组织、多租户、多法人、多站点隔离。
- 目标运行环境：Linux + Docker 单机部署；Windows 仅作为开发环境。
- 当前阶段不以高并发和复杂事务治理为重点，优先保证 PLM 业务功能、页面交互、对象关系和流程闭环合理完整。

## 1. 工作台与系统管理

- 个人工作台：待办审批、待处理变更、阶段门风险、近期任务。
- 组织机构：部门、角色、用户、岗位。
- 权限模型：菜单权限、对象权限、字段权限、流程权限。
- 数据字典：生命周期状态、文档分类、物料分类、变更类型、项目阶段。
- 属性模板：不同对象类型的字段模板和必填规则。
- 编码规则：产品、物料、BOM、文档、ECR、ECN、ECA、项目自动编码。
- 生命周期模板：设计中、流片、验证、试产、量产、冻结、废止。
- 工作流模板：文档签核、BOM 发布、ECR/ECO/ECN、项目阶段门。

## 2. 产品与物料主数据

- 产品型号台账：型号、内部料号、客户料号、工艺平台、晶圆尺寸、封装、等级、应用。
- 产品版本：版本、修订、状态、负责人、最近发布、资料完整度。
- 物料主数据：衬底/外延、Mask、光刻材料、镀膜材料、刻蚀气体、治具、供应商。
- 物料生命周期：新增、有效、替代、冻结、停用。
- 替代料与风险：替代料策略、供应商风险、关键料标识。
- 产品资料包：产品、BOM、工艺、文档、变更、质量数据聚合查看。

## 3. 文档管理

- 文档分类：规格书、工艺文件、测试报告、可靠性报告、设计资料、客户承认资料。
- 文档版本：版本、修订、状态、签核、发布、废止。
- 文档关联：关联产品、物料、BOM、工艺、变更、项目。
- 文档预览占位：PDF/Office/CAD 文件元数据与水印/签核状态。
- 文档变更：文档受 ECR/ECN 管控，发布后进入基线。

## 4. 需求规格管理

- 客户需求：客户规格、标准规范、质量体系输入。
- 需求追溯：需求到产品、BOM、工艺、文档、测试规范。
- 验收准则：每条需求有验收标准和状态。
- 阶段门需求：NPI 设计、流片、验证、试产、量产放行条件。

## 5. BOM 管理

- EBOM：研发工程 BOM，支持多层结构、用量、单位、工序位置、替代料。
- MBOM/PBOM 预留：后续扩展制造 BOM、工艺 BOM。
- BOM 版本：版本、状态、发布、冻结、废止。
- BOM 比较：版本差异、替代料差异、影响分析。
- BOM 导入导出：Excel 导入导出预留。
- BOM 基线：产品发布时冻结 BOM、文档、工艺和变更。

## 6. 工艺路线

- 工艺路线：外延来料、光刻、刻蚀、镀膜、清洗、量测、晶圆测试、切割封装、可靠性、OQC。
- 工艺参数：CD、Overlay、刻蚀深度、膜厚、片阻、LIV/IV、Wafer map。
- 工艺版本：版本、状态、负责人。
- 工艺与 BOM：材料和工序绑定。
- 工艺发布：ECN 生效后下发 MES 队列。

## 7. 工程变更

- ECR：变更申请、原因、变更前后、优先级、影响对象。
- 影响分析：产品、BOM、文档、工艺、在制 Lot、客户项目。
- 审批流程：研发、工艺、质量、项目、制造。
- ECN：变更通知、生效范围、生效批次。
- ECA：执行动作、责任部门、负责人、截止日期、执行结果。
- 下游同步：ERP/MES/QMS 轻量队列。

## 8. 项目管理

- NPI 项目：项目编号、型号、阶段、计划、负责人、风险。
- 阶段门：设计、流片、验证、试产、量产导入。
- 任务计划：任务、负责人、状态、截止日期。
- 项目交付物：BOM、文档、工艺、样品、可靠性报告。

## 9. 质量追溯

- Lot/Wafer：批次、晶圆、阶段、测试日期。
- 良率：CP、FT、Bin1、异常数量。
- 质量问题：问题编号、严重度、根因、纠正措施、状态。
- CAPA：质量闭环动作。
- 变更关联：质量问题可触发 ECR。

## 10. CAD/EDA 集成占位

- CAD/EDA 文件元数据：文件编号、版本、状态、关联产品。
- CAD BOM 导入占位：后续可接 AutoCAD/Creo/EDA 导入。
- 文件预览占位：不做本地 CAD 内核，仅保留文件与签核记录。

## 当前实现状态

- 已有后端数据模型/API：产品、物料、BOM、文档、工艺、变更、ECA、集成队列、项目、质量、需求、基线。
- 已有前端模块：驾驶舱、工作台、产品、需求、物料、BOM、基线、文档、工艺、变更、研产集成、项目、质量。
- 本轮目标：先让产品、物料、需求、文档具备真实 CRUD，并对接 SQLite 落库。
