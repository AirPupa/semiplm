# SemiPLM 商用化功能蓝图

## 对标原则

SemiPLM 面向半导体制造企业，产品能力按通用商用 PLM 的核心方法论设计，并结合现有 MES 制造建模页面校准业务边界。系统不复制第三方私有界面、文案和未公开实现，采用轻量级单机部署，优先实现产品定义、制造流程、用料表、版本关系、审批闭环和 MES 产品建模数据下发，CAD/EDA 深集成暂缓。

用户在迭代过程中的纠偏属于产品设计依据，必须持续沉淀到 `IMPLEMENTATION_ROADMAP.md` 和 `FUNCTION_LIST.md`。后续开发以这些文档为项目记忆：不合理的手输主数据、缺失 CRUD、演示口径文案、粗糙操作按钮和破坏表格密度的交互，都视为未完成。

部署目标为 Linux + Docker 单机运行。当前阶段保持架构轻量，优先补齐业务对象、版本关系、审批闭环和研产联动；不提前投入高并发、复杂事务、分布式、精细审计等工程治理，避免影响核心业务功能成型。

公开资料中，鼎捷 PLM 强调 PLM 作为研发数据源头，与 ERP、MES、SRM、APS、SCADA 串联；核心应用覆盖设计协同、产品结构 BOM、工程变更、项目、研产协同、工艺、产品配置。选型材料也强调电子行业 ECR/ECN 快速响应、自动触发文档更新、审批流程和生产计划调整。旧案例资料提到 CMII 式变更，包括问题报告 PR、ECR、ECN 与变更执行情况管理，并要求试产/打样、试产到量产监控、PLM 与 ERP 数据同步。

## 一期必须真实可用的核心对象

- 产品主数据：对齐 MES `ProductDef`，包含产品名称、产品版本、产品类型、生产类型、产品组、流程名称/版本、用料表名称/版本、Owner、Gross Die、Bank、光罩、Bin、Dummy 策略、状态和资料完整度。
- 产品制造定义：对齐 MES 产品页，制造流程详情页承载流程工序 `Seq`、流程内容 `Content`、动态参数 `Parameter`、量测 `Measure`、QTime、防污染 `Contamination`、动作 `Action`、分支 `Alter`，不单独建菜单。
- 物料主数据：衬底/外延、Mask、光刻材料、镀膜材料、刻蚀气体、测试治具、供应商、风险等级、生命周期。
- 需求规格：客户规格、NPI 阶段门、质量体系输入，追溯到型号、BOM、工艺、文档和测试规范。
- 文档管理：规格书、工艺流程卡、测试报告、可靠性报告、Mask layer 对照表，版本、状态、签核状态。
- BOM 管理：EBOM 起步，后续扩展 MBOM/PBOM/SBOM；支持用量、工序位置、替代料、状态。二期对齐 MES 用料表三段式（Consumable/Durable/Product）和工步绑定。
- 制造流程：PLM 主控源头，设计、审批、发布后同步给 MES；工序库作为受控步骤引用来源。
- 产品基线：冻结产品、BOM、工艺、文档、变更单，形成可发布数据包。
- 工程变更：ECR 提出、影响分析、审批、ECN 通知、ECA 执行动作、下游系统同步。
- NPI 项目：阶段、任务、负责人、计划日期、风险、进度。
- 质量追溯：Lot/Wafer、CP/FT 良率、Bin1、质量问题、根因、CAPA。

## 商用化路线

1. 数据主线
   建立“需求 -> 产品 -> 物料/BOM -> 工艺/文档 -> 基线 -> 变更 -> 质量”的可追溯链路。

2. 流程闭环
   工程变更从 ECR 申请开始，经过影响分析和签核，生成 ECN，并拆解 ECA 执行动作给工艺、质量、制造、物控、项目角色。

3. 研产协同
   已发布的 BOM、工艺、文档和变更结果进入 ERP/MES/QMS 同步队列。单机版只做队列和状态，不做真实外部系统连接。

4. 半导体行业化
   将通用制造 PLM 对象替换成 MES 可执行的半导体语言：ProductDef、工序序列、制程能力、制程配方、光罩/探针卡、抽检规则、QTime、污染等级、外延片、Mask、光刻 CD、刻蚀深度、膜厚、Wafer map、CP/FT。

5. UI 风格
   采用浅色企业后台、紧凑表格、详情抽屉/右侧面板、状态标签、阶段门和审批流。避免营销页、空卡片和纯演示图。
