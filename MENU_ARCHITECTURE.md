# SemiPLM 商用菜单与功能总清单

## 设计原则

- 以通用商用 PLM 方法论为主线，覆盖需求、产品、物料、文档、BOM、工艺、变更、项目、质量、集成和审计。
- 满足光电芯片制造行业场景，体现外延片、Mask、光刻、刻蚀、镀膜、清洗、量测、CP/FT、LIV/IV、可靠性等行业对象。
- 武汉睿码功能清单作为主要落地参考，鼎捷 PLM 作为辅助参考，只吸收项目管理、设计协同、产品结构、工程变更、研产协同等必要通用能力，不照搬其特有功能。
- 对照 Siemens Teamcenter、PTC Windchill、Aras Innovator、Dassault ENOVIA 等主流 PLM 的公开能力，优先实现通用 PLM 共性能力：单一产品数据源、BOM 多视图、制造工艺协同、工程变更闭环、文档与版本、项目阶段门、质量闭环和企业系统集成。
- 菜单按业务域分层，不按当前页面数量凑菜单；一个菜单只承载一个清晰业务对象或一组强相关对象。
- 组织、用户、角色、流程、基础配置必须拆分，不允许放在一个“大管理页”里。
- 主数据字段必须从主数据选择：人员、角色、产品、物料、文档、BOM、工艺路线、项目等都不能随意手输。
- 当前阶段先做轻量单机商用闭环：真实 CRUD、真实状态、真实关联、真实审批、真实集成队列；复杂 CAD/EDA 内核、复杂 ERP/MES 字段映射、高并发治理暂缓。
- UI 采用浅色企业后台、紧凑表格、右侧详情/抽屉、单行操作按钮；删除必须是小按钮并二次确认。

## 迭代顺序

1. 先按本文件调整左侧菜单结构，允许部分菜单先落到已有页面或占位，但不能做假功能。
2. 拆基础配置页：系统参数、编码规则已独立；后续继续拆分类属性、生命周期、数据字典。
3. 拆工程变更页：ECR、影响分析、ECA、版本归档先分视图，再逐步补 PR/ECO/ECN。
4. 深化工艺管理：工艺参数库、工序物料、工装辅料、检验标准。
5. 深化项目管理：模板、阶段门、计划、交付物。
6. 深化质量闭环：质量问题、CAPA、质量触发 ECR。
7. 补报表与审计。
8. 集成中心最后开始，当前只保持已有同步队列、接口端点和轻量回执能力，不优先扩展复杂映射、回调和监控。
